# ARMY Morning Brief

대한민국 육군 출근자를 위한 일일 공개 뉴스 브리핑 자동화 프로젝트입니다. Python 3.12와
`uv`를 사용하며, 공개된 HTTPS 출처에서 확인한 기사만 결정론적 규칙으로 수집·분류·중복
제거합니다. 이 저장소는 현재 설계와 패키지 뼈대만 포함합니다.

## 핵심 원칙

- 수집원은 로그인 없이 접근 가능한 공개 HTTPS 페이지와 피드로 제한합니다.
- 요약은 기사 제목·설명·본문에서 확인되는 문장을 발췌해 구성합니다. 여러 기사를 합쳐 새
  사실을 만들거나 전망을 생성하지 않습니다.
- 부대 별칭과 담당 지역은 코드가 아니라 향후 설정 파일에서 관리합니다.
- 부대 위치, 이동 경로, 병력 규모, 장비 배치 등 공개 기사에 없는 민감 정보를 추론하지
  않습니다.
- Telegram 자격 증명을 등록하기 전에 반드시 로컬 dry-run 결과를 검토합니다.

## 수집 시간 창

정기 실행은 매일 06:30 KST입니다. 실행 시각이 당일 07:00보다 이르므로 날짜 경계를
07:00로 잡지 않습니다. 기본 수집 구간은 **전날 14:00 KST부터 실제 실행 시각까지**이며,
모든 비교는 KST 기준의 시간대 인식 값으로 수행합니다. 수동 실행에서도 종료 시각은 고정
06:30이 아니라 실제 실행 시각입니다.

## 설정 설계

향후 설정은 다음 값을 코드 변경 없이 조정할 수 있어야 합니다.

- 사단별 정식 명칭과 검색 별칭(예: 숫자 표기, 한글 표기)
- 각 사단의 담당 지역 키워드와 행정구역
- 활성화할 공개 HTTPS 수집원과 출처 우선순위
- 카테고리별 최대 기사 수와 중복 판정 기준

별칭이나 지역이 일치한다는 이유만으로 군 관련 기사로 확정하지 않으며, 제목·출처·본문의
육군 업무 맥락을 함께 확인합니다.

## 개발 환경

필요 조건은 Python 3.12와 `uv`입니다.

```powershell
uv python install 3.12
uv sync --dev
Copy-Item .env.example .env
```

현재 뼈대의 정적 검증 명령은 다음과 같습니다. 테스트와 실행 모듈이 추가된 뒤 전체 명령을
통과시키는 것이 구현 완료 조건입니다.

```powershell
uv run pytest
uv run ruff check .
uv run ruff format --check .
uv run basedpyright
```

구현 단계에서 제공할 수집 전용 명령은 아래 형태로 고정합니다. `--dry-run`은 Telegram을
호출하지 않고 결과와 출처 URL만 로컬에 출력해야 합니다.

```powershell
uv run python -m army_morning_brief --dry-run
```

dry-run을 검토한 뒤에만 `.env` 또는 GitHub Actions Secrets에 아래 이름으로 값을
등록합니다. 실제 값은 문서, 코드, 로그, 커밋에 남기지 않습니다.

```text
TELEGRAM_BOT_TOKEN=<secret>
TELEGRAM_CHAT_ID=<secret>
```

제품 범위와 안전 기준은 [PRD.md](PRD.md), 구현 순서는 [TODOLIST.md](TODOLIST.md)에
정리되어 있습니다.

## 자동 실행 운영

GitHub Actions는 매일 06:30 KST(`30 21 * * *` UTC)에 예약됩니다. 예약 실행은 지연될 수
있고 공개 저장소 활동이 오래 없으면 비활성화될 수 있습니다. Secrets 설정, 수동 dry-run,
강제 재전송, 날짜별 중복 방지 한계와 현재 Telegram 자격 증명 인계 대기 상태는
[운영 문서](docs/OPERATIONS.md)를 따릅니다.
